1. PROJECT OVERVIEW
================================================================================

TeleSync is a SaaS-style social media automation platform that allows users to
schedule and auto-publish posts to multiple social platforms (Telegram channels,
Facebook Pages, and LinkedIn) using AI-generated captions. It integrates Google 
Gemini AI for content generation, external social platform APIs for message 
delivery, SMTP-based OTP email verification, and OAuth 2.0 login via Google 
and GitHub — all orchestrated through a custom background loop scheduler.

Problem Statement:
  Managing social media content manually is time-consuming and inconsistent.
  TeleSync solves this by allowing users to plan posts in advance, generate
  AI-powered captions, and have them published automatically at the right time
  across different social media networks — without any manual intervention.

8. CORE FEATURES — DETAILED
================================================================================

  8.1 Email OTP Verification
  ──────────────────────────
  - User submits registration form (name, email, password)
  - Backend creates unverified user, generates 6-digit OTP
  - OTP stored with 10-minute expiry in user record in database
  - Django Mail engine sends styled HTML email via Gmail SMTP
  - Frontend redirects to /verify-otp page with email in state
  - User enters 6-digit code in individual input boxes
  - Supports paste, backspace navigation, and 60s resend timer
  - On success: is_verified=true, otp cleared, JWT returned
  - Unverified users attempting login are redirected to OTP page

  8.2 OAuth 2.0 — Google & GitHub
  ────────────────────────────────
  - Login/Register pages show "Continue with Google" and "Continue with GitHub"
  - Clicking redirects browser to /api/auth/google or /api/auth/github
  - Django custom views handle OAuth redirection and token exchange flow
  - On callback: user looked up by email in PostgreSQL database
    * If exists: provider linked, is_verified set to true
    * If new: user created with avatar, provider info, is_verified=true
  - JWT token generated and passed via redirect to /oauth-success
  - OAuthSuccess page extracts token from URL, saves to localStorage
  - User redirected to dashboard seamlessly

  8.3 JWT Authentication
  ──────────────────────
  - JWT token expires in 7 days
  - Token stored in localStorage
  - Axios interceptor attaches Bearer token to every request
  - Global 401 handler auto-redirects to login and clears storage
  - Protected routes block unauthenticated access

  8.4 Post Scheduling
  ───────────────────
  - User creates a post with content, optional image URL, schedule time, and platforms
  - Validation: schedule time must be in the future
  - Post saved to PostgreSQL with status: "pending"
  - Django run_scheduler custom command runs in a background loop every minute
  - Queries all pending posts where scheduled_time <= now
  - Publishes to Telegram, Facebook, or LinkedIn depending on target platforms
  - Updates status to "posted" (or partial failure if some platforms failed)
  - On failure: status set to "failed" with error_message saved

  8.5 AI Caption Generation
  ─────────────────────────
  - User enters a topic keyword or selects a template prompt
  - Request sent to Google Gemini 2.0 Flash model
  - Uses x-goog-api-key header (required for AQ. prefix keys)
  - Prompt engineered for Telegram/Social formats:
      * Emoji-rich opening line
      * 2-3 engaging sentences
      * 5-7 relevant hashtags
      * Call to action
  - Falls back to gemini-2.0-flash-lite on 429/503 errors
  - Generated caption auto-fills the post textarea

  8.6 Telegram Integration
  ────────────────────────
  - sendMessage: plain text posts via /sendMessage
  - sendImage: photo posts via /sendPhoto with caption
  - HTML parse_mode enabled for formatting
  - Bot must be Admin of the target channel
  - Errors captured and stored per post in error_message field

  8.7 Dashboard & Posts Management
  ─────────────────────────────────
  - 4 stat cards: Total, Scheduled, Published, Failed
  - Recent 5 posts preview table on dashboard
  - Full posts list (Scheduler page) with filter tabs: All / Pending / Posted / Failed
  - Connected account management via Accounts page
  - Delete any post, refresh button, post count display

9. SECURITY MEASURES
================================================================================

  - Passwords never stored in plaintext (bcrypt hashing)
  - Email must be OTP-verified before login allowed
  - OAuth users auto-verified (trusted provider)
  - JWT tokens expire after 7 days
  - All post/AI/social routes protected by JWTAuthentication handler
  - Posts scoped to user_id (users cannot access others' posts)
  - .env file gitignored — credentials never committed
  - CORS restricted/allowed via django-cors-headers configurations
  - OTP expires in 10 minutes, cleared after use
  - Input validation on all API endpoints

11. INSTALLATION & SETUP
================================================================================

  Prerequisites:
    - Python 3.10+
    - PostgreSQL
    - Node.js v18+
    - Telegram Bot Token (from @BotFather)
    - Google Gemini API Key (from aistudio.google.com)
    - Gmail App Password (for SMTP OTP)
    - Google OAuth credentials (console.cloud.google.com)
    - GitHub OAuth credentials (github.com/settings/developers)

  Step 1: Clone repository
    git clone <repo-url>
    cd Social_Media_Automation

  Step 2: Setup backend
    cd backend
    python -m venv backend_env
    source backend_env/bin/activate
    pip install -r requirements.txt
    python manage.py migrate
    # Create or update .env file with your PostgreSQL and API credentials
    python manage.py runserver &
    python manage.py run_scheduler &

  Step 3: Setup frontend
    cd client
    npm install
    npm run dev

  Step 4: Open browser
    http://localhost:5173

  Google OAuth Setup:
    - Authorized JavaScript Origins: http://localhost:5173
    - Authorized JavaScript Origins: http://localhost:8000
    - Authorized Redirect URI: http://localhost:8000/api/auth/google/callback

  GitHub OAuth Setup:
    - Homepage URL: http://localhost:5173
    - Authorization callback URL: http://localhost:8000/api/auth/github/callback

12. CHALLENGES FACED & SOLUTIONS
================================================================================

  Challenge 1: CORS errors on API calls
  → Fixed by adding Vite proxy (/api → localhost:8000)
    and configuring django-cors-headers in settings.py

  Challenge 2: Gemini API model not found (1.5-flash unavailable)
  → Queried available models via ListModels API
    Updated to use gemini-2.0-flash and gemini-2.0-flash-lite

  Challenge 3: Gemini API key format (AQ. prefix vs AIzaSy)
  → AQ. keys require x-goog-api-key header instead of ?key= query param
    Updated service to use header-based authentication

  Challenge 4: IPv6 vs IPv4 routing issues under certain network setups
  → Enforced IPv4 socket connections in settings.py to bypass IPv6 network routing issues

  Challenge 5: Telegram posting to bot instead of channel
  → TELEGRAM_CHANNEL_ID was set to bot username
    Must be set to actual channel username (@channel)
    Bot must be added as Admin to that channel

  Challenge 6: Django scheduler loop stability
  → Managed background processes running python manage.py run_scheduler
    Added robust exception handling to keep scheduler alive on API failures

  Challenge 7: Google OAuth — invalid origin error
  → Authorized JavaScript Origins must not contain paths
    Correct: http://localhost:5173 (no trailing slash, no path)

  Challenge 8: GitHub OAuth — wrong Homepage URL
  → Homepage URL must be frontend URL (http://localhost:5173)
    Not the callback URL
